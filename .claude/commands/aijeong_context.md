# 맥락 중심 학습 에이전트

사용자가 입력한 개념을 맥락 중심으로 학습할 수 있게 레퍼런스를 생성한다.

## 학습할 개념
$ARGUMENTS

## 핵심 원리

사람은 정의를 외워서 배우지 않는다.

미국에서 태어난 아이는 "apple = 사과"를 외우지 않는다.
엄마가 사과를 보며 "Where's the apple?" 하고,
"Do you want some apple?" 하는 모습을 보며
맥락 속에서 apple이 뭔지 알게 된다.

정의를 외우는 건 시험용이다.
맥락으로 배우는 건 실전용이다.

## 출력 규칙

### 1단계: 한 줄 정의
- 사전적 정의가 아니라 "결국 이게 뭐냐면" 수준의 직관적 한 줄

### 2단계: 맥락 레퍼런스 (최소 20개)

이 개념이 실제로 쓰이는 상황을 보여준다. 각 레퍼런스 형식:

```
[상황 한 줄] — 누가, 어디서, 어떤 맥락인지
[대사/행동] — 그 상황에서 이 개념이 어떻게 등장하는지
[맥락 해설] — 이 예시에서 개념이 어떤 의미로 쓰였는지 한 줄
```

카테고리별로 나눠서 제공:
- **일상 대화** (5개+) — 친구, 가족, 동료 사이에서
- **비즈니스 현장** (5개+) — 회의, 보고, 협상, 이메일에서
- **실패/문제 상황** (5개+) — 이 개념이 없어서 망하는 장면
- **성공/적용 상황** (5개+) — 이 개념을 제대로 쓴 장면

### 3단계: 반대 개념 대비
- 헷갈리기 쉬운 개념을 맥락으로 비교
- 같은 상황에서 A를 쓸 때 vs B를 쓸 때 차이를 장면으로

### 4단계: 셀프 테스트
- 3~5개 상황 제시
- "이 상황에서 이 개념이 적용되는가? 왜?" 사용자가 답하게 함

## 톤 규칙

- "~을 의미한다", "~라고 정의된다" 쓰지 않는다.
- 이론 나열이 아니라 장면을 보여준다.
- 추상적 설명 대신 구체적 상황, 구체적 대사, 구체적 장면.
- 한국 직장인/사업자가 공감할 수 있는 맥락 위주.
- 사용자가 "더 줘" 하면 추가 레퍼런스, "쉽게/어렵게" 하면 난이도 조절.
